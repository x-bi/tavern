# 阶段 49｜回归测试与修复

## 快速读取

- 波次: 部署、备份、回归与 MVP 验收
- 阶段目标: 对 MVP 做系统级回归清单与 bug 修复。
- 前置依赖: 阶段 48。
- 文件范围: tests/* 、修复点相关文件。
- 本阶段要做: 端到端关键路径清单、手工 + 自动化基础检查。
- 本阶段不要做: 不做庞大测试平台。
- 验收标准: 1. 核心路径都有测试或手工回归记录 2. 阻塞级问题已修复 3. 已知问题有文档化清单
- 完成后输出: 1. 修改文件列表 2. 新增文件列表 3. 回归清单 4. 修复内容 5. 未解决问题 6. 风险和 TODO
